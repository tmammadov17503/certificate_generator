# Certificate Claim App

This app creates one-time certificate links. Each link can generate one PDF certificate only once.

## What it does

- Lets you create unique claim links from the dashboard
- Shows a certificate page where the participant enters a name
- Generates the PDF using your uploaded certificate template
- Marks the link as used immediately after a successful claim
- Blocks all later attempts with the same link

## Run locally

```powershell
cd C:\Users\ASUS\Documents\Apps\oracle\certificate-claim-app
python app.py
```

Then open:

```text
http://127.0.0.1:5050
```

## Main folders

- `static/certificate-template.png`: your certificate template
- `instance/certificates.db`: SQLite database with issued and used links
- `output/`: archived PDF copies generated by the app

## Optional environment variables

- `CERTIFICATE_HOST`: server host, default `127.0.0.1`
- `CERTIFICATE_PORT`: server port, default `5050`
- `PORT`: hosting platform port override, used automatically when present
- `CERTIFICATE_BASE_URL`: public URL for generated links, useful when deployed
- `CERTIFICATE_FONT_PATH`: override font path if you do not want the default Calibri lookup
- `CERTIFICATE_DATA_DIR`: persistent folder for SQLite and archived PDFs, useful on Railway/Render volumes
- `CERTIFICATE_ADMIN_PASSWORD`: password protecting the dashboard when the app is public
- `CERTIFICATE_APP_SECRET`: Flask session secret for admin login

## Notes

- The app blocks reuse of the same link, but it cannot stop someone from sharing the PDF after downloading it.
- When `CERTIFICATE_ADMIN_PASSWORD` is set, the organizer dashboard requires login.

## Deployment direction

The easiest hosted path for this app is a Python platform that supports:

- a public web service URL
- a persistent disk or volume
- environment variables

This app is already prepared for that model.
